## 1. Проблема

Команда из 2 senior fullstack-разработчиков (Денис, Imei Rhen) и 1 product-менеджера (Кирилл) работает на фриланс-площадке Профи.ру. Текущий процесс полностью ручной:

1. Кирилл мониторит Профи.ру, копирует заявки в TG-чат (#24.02.26_1, #24.02.26_2...)
2. Разработчики читают каждую заявку, ставят +/- и пишут сроки/цены
3. Кирилл формирует отклик вручную и отправляет на площадке

**Боли:**
- Кирилл тратит целые дни на ручной мониторинг и копирование заявок
- Разработчики тратят время на чтение нерелевантных заявок (WordPress, обучение ИИ и т.д.)
- Нет стандартизированного формата оценки (кто-то пишет "+, 5-6 суток", кто-то "80к")
- Отклик формируется вручную каждый раз заново
- Заявки теряются в потоке чата

## 2. Решение

**DevGrabBot** — Telegram-бот с AI-ассистентом, который автоматизирует полный цикл: парсинг заявок → AI-анализ → оценка разработчиками → формирование отклика для менеджера.

## 3. Архитектура

### 3.1 Подход: модульная архитектура с очередью

```
┌─────────────────────────────────────────────────────────────────┐
│                     DevGrabBot System                           │
├─────────────┬──────────────┬──────────────┬────────────────────┤
│   Parser    │   AI Engine  │   TG Bot     │   Database         │
│  (Profiru)  │  (OpenRouter)│  (aiogram 3) │  (PostgreSQL)      │
│             │              │              │                    │
│ Selenium/   │ Анализ       │ Групповой    │ orders             │
│ BS4         │ заявки →     │ чат: обзор   │ ai_analyses        │
│ ↓           │ оценка       │ заявок       │ order_assignments  │
│ Фильтрация  │ стека/цены/  │              │ manager_responses  │
│ Дедупликация│ сроков       │ Личка: кнопки│ team_members       │
│ ↓           │              │ редактиров.  │ settings           │
│ Redis Queue │              │ утверждение  │                    │
└──────┬──────┴──────┬───────┴──────┬───────┴────────────────────┘
       │             │              │
       └─────────────┴──────────────┘
              Redis (очередь + кэш)
```

Каждый модуль — независимый процесс, связь через Redis-очередь. Паттерн аналогичен WB Price Bot.

### 3.2 Деплой

- **VPS** + **venv** (виртуальное окружение Python)
- **systemd** сервисы для каждого модуля (bot, parser-worker, scheduler)
- PostgreSQL и Redis — установлены на VPS
- Без Docker

## 4. Поток обработки заявки

```
1. ПАРСИНГ (cron / scheduler, каждые 5-15 мин)
   Parser → парсит Профи.ру → фильтрует по ключевым словам
   → дедупликация → новая заявка → Redis queue "new_orders"
   Основа: https://github.com/dobrozor/parser_profiru (Selenium + BS4)

2. AI-АНАЛИЗ (автоматически, worker)
   AI Engine подхватывает из очереди → OpenRouter API
   → Анализ: стек, цена, сроки, релевантность
   → Результат → БД (статус "pending_review")

3. УВЕДОМЛЕНИЕ РАЗРАБОТЧИКАМ (групповой чат)
   ┌──────────────────────────────────────┐
   │ Новая заявка #24.02.26_9            │
   │                                      │
   │ ВЫЖИМКА:                             │
   │ TG-бот для визового центра.          │
   │ Мониторинг VFS/TLS/BLS, уведомления, │
   │ автозапись на слоты.                  │
   │                                      │
   │ СТЕК: Python, Selenium, aiogram      │
   │ ЦЕНА: 45 000 руб.                    │
   │ СРОКИ: 10-14 дней                    │
   │ РЕЛЕВАНТНОСТЬ: 92%                   │
   │                                      │
   │ [Взять] [Пропустить]                 │
   └──────────────────────────────────────┘

4. РАЗРАБОТЧИК БЕРЁТ ЗАЯВКУ (нажимает "Взять")
   → Бот пишет в личку разработчику:
   ┌──────────────────────────────────────┐
   │ Заявка #24.02.26_9 — ваша!          │
   │                                      │
   │ [Полный AI-анализ...]                │
   │                                      │
   │ [Изменить цену]  [Изменить сроки]   │
   │ [Изменить стек]  [Своё сообщение]   │
   │                                      │
   │ [Утвердить и отправить менеджеру]    │
   └──────────────────────────────────────┘

5. РЕДАКТИРОВАНИЕ (опционально)
   Разработчик нажимает кнопку → вводит правку
   → AI перегенерирует сообщение с учётом правки
   → Показывает обновлённый вариант → снова кнопки

6. УТВЕРЖДЕНИЕ → МЕНЕДЖЕРУ
   Разработчик нажимает "Утвердить" →
   AI формирует готовый отклик для площадки:
   ┌──────────────────────────────────────┐
   │ Кириллу:                             │
   │                                      │
   │ Заявка #24.02.26_9 утверждена!       │
   │ Исполнитель: @Web_Dusha             │
   │                                      │
   │ ГОТОВЫЙ ОТКЛИК:                       │
   │ "Здравствуйте! Имеем опыт работы    │
   │ с парсингом визовых сайтов и TG-     │
   │ ботами. Предлагаем решение на        │
   │ Python + Selenium..."                │
   │                                      │
   │ 45 000 руб. | 10-14 дней            │
   │                                      │
   │ [Скопировать отклик]                 │
   └──────────────────────────────────────┘
```

## 6. AI-промпты

### 6.1 Промпт анализа заявки (для разработчиков)

**Системный промпт:**
> Ты — аналитик IT-команды из 2 senior fullstack-разработчиков. Стек команды: Python, JavaScript/TypeScript, React/Next.js, Telegram-боты (aiogram), AI/ML интеграции, парсеры (Selenium, BS4), PostgreSQL, Redis. Команда НЕ берёт: WordPress, обучение, вакансии на полный рабочий день, проекты без ТЗ с бюджетом до 10 000 руб.

**Вход:** raw_text заявки

**Выход (JSON):**
```json
{
  "summary": "Краткая выжимка (2-3 предложения)",
  "stack": ["Python", "Selenium", "aiogram"],
  "price_min": 35000,
  "price_max": 50000,
  "timeline_days": "10-14",
  "relevance_score": 92,
  "complexity": "medium",
  "response_draft": "Черновик предложения..."
}
```

### 6.2 Промпт формирования отклика (для менеджера)

**Системный промпт:**
> Ты — копирайтер IT-команды. Формируй профессиональный отклик на заявку для фриланс-площадки. Стиль: конкретный, без воды, с упоминанием релевантного опыта. Отклик должен быть готов к копированию и отправке.

**Вход:** утверждённый анализ + правки разработчика

**Выход:** готовый текст отклика

## 9. Внешние зависимости

- **OpenRouter API** — ключ для доступа к LLM (конфигурируемая модель)
- **Профи.ру** — аккаунт для парсинга (логин/пароль)
- **Telegram Bot API** — токен от @BotFather
- **VPS** — сервер с PostgreSQL, Redis, Python 3.11+, Chrome/Chromium (для Selenium)

## 10. Риски и ограничения

| Риск | Митигация |
|------|-----------|
| Бан аккаунта на Профи.ру | Настраиваемые интервалы парсинга, ротация User-Agent, human-like задержки |
| Изменение HTML Профи.ру | Модульный парсер — легко обновить селекторы |
| Качество AI-анализа | Итеративная доработка промптов, разработчик всегда может поправить |
| Стоимость OpenRouter | Конфигурируемая модель — можно переключиться на более дешёвую |
